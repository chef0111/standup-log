## Post-phase polish ‚Äî floating tabs

Follow-up after Phase 8‚Äì9: tab bar appeared at **top** on web (Expo‚Äôs default NativeTabs web UI) and could collapse to top pill on iOS when scrolling (`minimizeBehavior`).

### Root causes

1. **Web:** `NativeTabs` ‚Üí `NativeTabsView.web` renders a fixed **top** pill (`native-tabs.module.css`: `top: 24px`). Need platform-specific layout.
2. **Native:** `minimizeBehavior="onScrollDown"` (or `automatic`) moves tab bar to top while scrolling on iOS 26+.
3. **Scroll overlap:** Bottom CTAs (Save / Regenerate on Standup) needed extra padding above tab bar.

### What we did

| File                                                                   | Purpose                                                                                                                              |
| ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `src/features/shell/components/app-tab-layout.tsx`                     | Native: `minimizeBehavior="never"`, `blurEffect="systemMaterial"` (iOS), `backgroundColor=popover`, `disableTransparentOnScrollEdge` |
| `src/features/shell/components/app-tab-layout.web.tsx`                 | Web: `expo-router/ui` bottom **floating pill** (`bg-popover`, rounded, shadow), icons + labels                                       |
| `src/features/shell/lib/tab-items.ts`                                  | Shared tab config (names, hrefs, SF/md icons, Lucide for web)                                                                        |
| `src/features/shell/lib/tab-bar-metrics.ts`                            | `FLOATING_TAB_BAR_HEIGHT`, `FLOATING_TAB_BAR_MARGIN`, `getFloatingTabBarInset()`                                                     |
| `src/features/shell/hooks/use-tab-bar-scroll-padding.ts` (+ `.web.ts`) | Scroll bottom padding on all tab screens                                                                                             |
| `src/features/theme/hooks/use-theme-color.ts`                          | Extended `ThemeColorVariable` for tab tokens (`--color-popover`, etc.)                                                               |

**Commit:** `e295ec3 feat(ui): implement floating native tabs`

### Platform expectations

- **iOS 26+:** System liquid-glass floating tab bar at bottom (native `NativeTabs`).
- **Android:** Material 3 bottom nav via `NativeTabs` ‚Äî shape is system-controlled (not a custom floating pill).
- **Web:** Custom bottom floating capsule via `app-tab-layout.web.tsx` (Metro resolves `.web.tsx` automatically).

### TypeScript fix (web width)

React Native `ViewStyle.width` does not accept CSS `min()`. Use `className="w-[92%] max-w-[420px]"` instead of `width: 'min(420px, 92vw)'` in `app-tab-layout.web.tsx`.
